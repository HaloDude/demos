After installing the chart, please configure Prometheus datasource named 'prometheus' to make all dashboards work.

There is a bug in current grafana version (4.4.1 as of writing this readme) which causes dashboards to be duplicated.

Prometheus URLs:
{{- if .Values.prometheus.ingress.domain }}
https://{{ .Values.prometheus.ingress.domain }}/
{{- else if .Values.kubernetesApiEndpoint }}
- {{ .Values.kubernetesApiEndpoint }}/api/v1/namespaces/{{ .Release.Namespace }}/services/{{ .Release.Name }}-prometheus/proxy/
{{- end }}
{{- if .Values.grafana.enabled }}
Grafana URLs:
{{- if .Values.grafana.ingress.domain }}
- https://{{ .Values.grafana.ingress.domain }}
{{- else if .Values.kubernetesApiEndpoint }}
- {{ .Values.kubernetesApiEndpoint }}/api/v1/namespaces/{{ .Release.Namespace }}/services/{{ .Release.Name }}-grafana/proxy/
{{- end }}
{{- end }}
{{- if .Values.alertmanager.enabled }}
Alertmanager URLs:
{{- if .Values.alertmanager.ingress.domain }}
- https://{{ .Values.alertmanager.ingress.domain }}/
{{- end }}
{{- if .Values.kubernetesApiEndpoint }}
- {{ .Values.kubernetesApiEndpoint }}/api/v1/namespaces/{{ .Release.Namespace }}/services/{{ .Release.Name }}-alertmanager/proxy/
{{- else }}
- <kubernetes-api-endpoint>/api/v1/namespaces/{{ .Release.Namespace }}/services/{{ .Release.Name }}-alertmanager/proxy/
{{- end }}
{{- end }}
